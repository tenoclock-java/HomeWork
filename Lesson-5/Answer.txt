1.Произвести ручную настройку сети в Ubuntu, на каждом шаге сделать скриншоты.
	Оба варианта настройки представлены в файле 00-installer-config.yaml

2.Переключить настройку сети на автоматическую через DHCP, проверить получение адреса.
	Вариант с dhcp был выбран при установке и закомментирован в файле 00-installer-config.yaml

3.Изменить адрес DNS на 1.1.1.1 и проверить доступность интернета, например, открыв любой браузер на адрес https://geekbrains.ru.
	Для текущей сессии можно изменить адрес в файле /etc/resolv.conf и проверить работоспособность с помощью утилиты dig
	
4.* Настроить правила iptables, чтобы из внешней сети можно было обратиться только к портам 80 и 443. Запросы на порт 8080 перенаправлять на порт 80.
	Настройки в файле iptables.rules

5.* Дополнительно к предыдущему заданию настроить доступ по ssh только из указанной сети.
	Настройки в файле iptables.rules

6.* Настроить OpenVPN, связать несколько виртуальных машин с помощью OpenVPN-туннеля. 
	Конфиг сервера в файле server.conf, настройка клиента в файле aws_vpn_client.ovpn (использовал клиент openVPN для Windows и подключался со своего ноутбука)
	Т.к. за сервером по сути нет никаких машин то подсеть для доступа состоит из 1 адреса - это сам сервер.
	Чтобы сервер не блокировал пакеты из сети VPN надо добавить 2 правила:
	 
	 iptables -A FORWARD -d 192.168.180.0/24 -s 172.31.38.8/32 -j ACCEPT
	 iptables -A FORWARD -s 192.168.180.0/24 -d 172.31.38.8/32 -j ACCEPT
	 Так же разрешить соединение для доступа к порту openVPN:
	 iptables -A INPUT -p TCP --dport 1194 -j ACCEPT (я использую в конфигах именно TCP чтобы не было потерь до амазона)
	Еще требуется включить форвардинг пакетов на сервере:
		echo 1 > /proc/sys/net/ipv4/ip_forward,  
		а в файле /etc/sysctl.conf раскомментировать строку net.ipv4.ip_forward=1
		

7.* Сделать одну из настроенных в задании выше машин шлюзом доступа в интернет. Настроить NAT.
	Для того чтобы все пакеты после установления VPN-соеднинения шли через сервер необходимо добавить 2 строчки в конфигурационный файл сервера:
	push "redirect-gateway def1"
	push "dhcp-option DNS 77.88.8.8"
	Т.к. NAT осуществляется средствами Amazon (сервер не имеет прямого внешнего адреса), то правило MASQUERADE на сервере добавлять не имеет смысла.
